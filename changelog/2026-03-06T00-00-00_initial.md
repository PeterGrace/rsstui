# rsstui — Initial Implementation

**Date:** 2026-03-06
**Version:** 0.1.0

## Overview

First complete implementation of `rsstui`, a terminal-based RSS/Atom feed reader
built with `ratatui` and `tokio`.

## Architecture

### Module layout

| File | Responsibility |
|---|---|
| `src/main.rs` | Entry point: tracing bootstrap, terminal setup/teardown, panic hook, async event loop |
| `src/error.rs` | `AppError` enum covering HTTP, parse, IO, serde, and terminal failures |
| `src/feed.rs` | `Article` / `FeedData` domain types; `fetch_feed` async function |
| `src/storage.rs` | `StorageConfig` / `FeedConfig`; atomic load/save to `~/.local/share/rsstui/feeds.json` |
| `src/app.rs` | `App` struct; all keyboard event handling; background task coordination via `tokio::sync::mpsc` |
| `src/ui.rs` | All ratatui rendering: three-pane layout, modals, status bar |

### Async model

The event loop runs inside `#[tokio::main]` and uses `tokio::select!` over:

1. `crossterm::event::EventStream` — async stream of terminal events (key press, resize).
2. `tokio::time::interval` (100 ms) — periodic tick that drains results from
   background fetch tasks via `try_recv`.

Feed fetches are spawned as `tokio::task` futures.  Results flow back through an
`mpsc::UnboundedChannel<FeedMessage>`.  This keeps the TUI completely responsive
while fetches are in flight.

### Persistence

Config is stored as pretty-printed JSON at `$XDG_DATA_HOME/rsstui/feeds.json`
(falling back to `~/.local/share/rsstui/feeds.json`).  Each feed entry records
its URL and the set of article IDs the user has marked as read.  Writes are
atomic: a temporary file is written then renamed to the final path.

## UI Layout

```
┌──────────────────────────────────────────────────────────────────┐
│ rsstui  RSS Feed Reader                       [fetching...]      │  title bar
├─────────────────┬──────────────────────┬────────────────────────┤
│ Feeds           │ Articles             │ Preview                │
│                 │                      │                        │
│ > Hacker News   │ * Breaking news      │ Breaking news title    │
```

## Keybindings

| Key | Action |
|---|---|
| `Tab` / `Shift-Tab` | Cycle focus between Feeds / Articles / Preview panes |
| `j` / `Down` | Move selection down |
| `k` / `Up` | Move selection up |
| `g` | Jump to top of list |
| `G` | Jump to bottom of list |
| `Enter` | Advance focus: Feeds -> Articles -> Preview |
| `a` | Open "Add Feed" modal |
| `d` | Open "Confirm Delete" modal (Feeds pane) |
| `r` | Refresh selected feed |
| `R` | Refresh all feeds |
| `m` | Toggle read/unread on selected article |
| `o` | Open article URL in system browser (`xdg-open` / `open`) |
| `u` / `d` | Scroll preview up/down 5 rows |
| `q` / `Ctrl-C` | Quit |

## Dependencies added

| Crate | Purpose |
|---|---|
| `ratatui 0.29` | TUI rendering framework |
| `crossterm 0.28` (event-stream) | Terminal backend + async event stream |
| `reqwest 0.12` (rustls-tls) | Async HTTP client |
| `feed-rs 2.3` | RSS 0.9x/1.0/2.0, Atom 0.3/1.0, JSON Feed parsing |
| `futures 0.3` | `StreamExt` for `EventStream::next()` |
| `chrono 0.4` | Timestamp types and formatting |
| `serde / serde_json` | Config serialization |
| `anyhow / thiserror` | Error handling |
| `dotenvy 0.15` | `.env` loading |
| `tokio 1` (rt-multi-thread) | Async runtime |
| `tracing / tracing-subscriber` | Structured logging |

## Removed from scaffold

- `ctrlc` — Ctrl-C handled via keyboard event matching
- `lazy_static` — no global statics needed
- `console-subscriber` — tokio-console debugging not needed in production
- `tracing-log` — not needed without console-subscriber
- `dotenv` — replaced with `dotenvy` per project guidelines
