## Summary

RSS feed summaries and content now render with visual structure instead of
showing raw HTML tags.  A two-stage pipeline converts HTML to styled terminal
output:

```
raw HTML  ‚îÄ‚îÄ(htmd)‚îÄ‚îÄ‚ñ∫  Markdown  ‚îÄ‚îÄ(pulldown-cmark)‚îÄ‚îÄ‚ñ∫  ratatui Text
```

## Changes

### New: `src/markdown.rs`

Owns two public functions:

| Function | Input | Output |
|---|---|---|
| `html_to_markdown(html)` | raw HTML fragment | Markdown `String` |
| `render_markdown(md)` | Markdown `&str` | `ratatui::text::Text<'static>` |

**`html_to_markdown`** delegates to the `htmd` crate, which uses `html5ever`
to correctly parse malformed HTML and emits clean Markdown.  Handles `<br>`,
`<h1>`‚Äì`<h6>`, `<p>`, `<strong>`/`<b>`, `<em>`/`<i>`, `<a>`, `<ul>`/`<ol>`,
`<li>`, `<code>`, `<pre>`, `<blockquote>`, `<hr>`, and more.

**`render_markdown`** drives a `pulldown-cmark` `Parser` event loop and builds
up `Vec<Span<'static>>` lines with these styling rules:

| Element | ratatui Style |
|---|---|
| H1 | Cyan + Bold |
| H2 | Blue + Bold |
| H3 | Green + Bold |
| H4‚ÄìH6 | Bold |
| `**strong**` | Bold (additive with parent) |
| `*emphasis*` | Italic (additive with parent) |
| `~~strikethrough~~` | Crossed-out |
| `` `inline code` `` | Green |
| ` ```code block``` ` | Green, bordered with `‚îÄ` rules |
| Unordered list items | `‚Ä¢` bullet in Yellow |
| Ordered list items | numbered `1.`, `2.`, ‚Ä¶ |
| Blockquote | `\|` gutter, DarkGray Italic |
| Link text | Cyan + Underlined |
| Images | `[image]` placeholder (DarkGray) |
| Horizontal rule | `‚îÄ` √ó 40 in DarkGray |

### Modified: `src/feed.rs`

`fetch_feed` now calls `html_to_markdown()` on the raw summary/content string
at fetch time.  `Article::summary` therefore stores Markdown, not HTML.
Conversion happens once per article, not on every render frame.

### Modified: `src/ui.rs` (`render_preview_pane`)

The preview pane no longer builds `Line` vectors manually.  Instead it
constructs a small Markdown document that includes the article metadata
(title escaped against Markdown special chars, published date, link) and
appends `article.summary`, then calls `render_markdown()` to produce the
styled `Text` for the `Paragraph` widget.

A new private helper `escape_md(text)` prevents article titles with
characters like `*`, `_`, or `[` from accidentally triggering Markdown
formatting in the header block.
